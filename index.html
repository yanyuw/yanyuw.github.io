fgets函数

第一个参数表示存储所得数据的数组；
第二个参数表示指明存储数据的大小;（若为n，则从流中读入n-1个字符 fgets空出一个数组元素为’\0’准备）
第三个参数表示将要读取的文件流（stdin表示读取键盘输入的数据）；

fgets会将行末的换行符(‘\n’)读入，导致数组长度会+1

有遇到fgets将我不需要的第一行数据读入了数组的情况，这时候可以再用一个fgets函数将不需要的数据先读入一个数组，再用fgets读取真正需要的数据。

跳过不需要的数据

读取字符可能会把空格读取进去 用scanf("%d%*c%c",&n，&c); 可以避免把空格赋值给变量

“*”符:用以表示该输入项,读入后不赋予相应的变量，即跳过该输入值。

例如：输入1 2时 若不用*符，则n为1，c为空格；用*符后 跳过了空格，所以n为1，c为’2’；

字符串相关的函数

strlen(a) 取字符串长度：求字符串a的长度（不包括结束符(‘\0’)）

strcpy(a, b) 复制字符串：将b拷贝给a

strncpy(a, b, n) 复制指定长度字符串：截取b的n个字符拷贝给a

strcat(a, b) 附加字符串：在字符串a之后连接上b

strncat(a, b, n) 附加指定长度字符串：从b截取n个字符连接在字符串a之后

strcmp(a, b) 比较字符串：根据ASCII码逐个字符比较字符串a、b 当a < b 时 返回负数 当a > b 时 返回正数 当a == b 时返回0

strcasecmp(a,b) 忽略大小写比较字符串

strncmp(a, b, n) 比较指定长度字符串 最多比较n个字符

strchr(a, c) 从字符串头开始在字符串a中查找指定字符c首次出现的位置 返回指向第一次出现c位置的指针 没有则输出null

strrchr(a, c) 从字符串尾开始在字符串a中查找指定字符c首次出现的位置

strstr(a, b) 查找字符串：在字符串a中查找字符串b首次出现的位置 没有则返回null 有则从首次发现b字符串处返回a字符 （a的长度必须大于b）

从输入 URL 到页面加载完的过程中都发生了什么事情？

输入URL

浏览器会接受用户所输入的url并解析 根据格式分成几个部分

浏览器通过域名 向DNS寻求 找出其对应的IP地址

浏览器向确定的IP和端口号发起网络连接

浏览器根据协议（http）要求，向服务器组织一个请求的数据包

里面包含大量请求信息，包括请求的资源路径、你的身份

网站服务的重定向响应

（如fackbook的永久重定向相应 这样浏览器就会访问“http://www.facebook.com/” 而非“http://facebook.com/”。 重定向：将带3w和不带3w归于同一网站 与搜索引擎排名有关）

浏览器跟踪重定向地址

服务器“处理”请求，将数据返回给浏览器。

数据可能是页面的布局、文字。数据也可能是图片、脚本程序等。若资源路径指示的资源不存在，服务器就会返回404错误。

浏览器开始显示HTML

浏览器发送获取嵌入在HTML中的对象的请求

（根据页面里一些外链的URL，例如图片的地址，按照以上步骤再次发送一个获取请求来重新获得这些文件）

浏览器开始渲染，根据资源的类型，将资源组织成屏幕上显示的图像

（浏览器将HTML变成人眼看到的图像的过程）

浏览器将渲染好的页面图像显示出来，并开始响应用户的操作。浏览器发送异步（AJAX）请求

附：部分概念

URL：

统一资源定位符（或称统一资源定位器/定位地址、URL地址等，俗称为网页地址（网址）；英语：Uniform Resource Locator，缩写URL）如同在网络上的门牌，是因特网上标准的资源的地址。

统一资源定位符的标准格式如下：
　　协议类型:[//服务器地址[:端口号]][/资源层级UNIX文件路径]文件名[?查询][#片段ID]

统一资源定位符的完整格式如下：
　　协议类型:[//[访问资源需要的凭证信息@]服务器地址[:端口号]][/资源层级UNIX文件路径]文件名[?查询][#片段ID]

其中【访问凭证信息@；:端口号；?查询；#片段ID】都属于选填项。

例如：http://www.guokr.com/question/554991/
协议部分：http （从该计算机获取资源的方式，常见的是HTTP、FTP，不同协议有不同的通讯内容格式）
网络地址：www.guokr.com （指示该连接网络上哪一台计算机，可以是域名或者IP地址，可以包括端口号）
资源路径：/question/554991/ （指示从服务器上获取哪一项资源。）

DNS：

域名系统（英文：Domain Name System，缩写：DNS）是互联网的一项服务。它作为将域名和IP地址相互映射的一个分布式数据库，能够使人更方便地访问互联网。DNS使用TCP和UDP端口53

端口：

逻辑端口是指逻辑意义上用于区分服务的端口，如TCP/IP协议中的服务端口，端口号的范围从0到65535，比如用于浏览网页服务的80端口，用于FTP服务的21端口等。由于物理端口和逻辑端口数量较多，为了对端口进行区分，将每个端口进行了编号，这就是端口号。
主机通过“IP地址+端口号”来区分不同的服务。

渲染：

构建渲染树，对各个元素进行位置计算、样式计算等等，然后根据渲染树对页面进行渲染（可以理解为“画”元素）

渲染大致分为两部分，一部分是排版，一部分是绘制。排版就是根据文档流，加上浮动、定位等属性，确定各个盒子的位置还有尺寸。绘制就是将css的属性例如字体、背景色、圆角等等按照给定的方式呈现出来。各个浏览器的渲染效率不一样。

域名和url

域名就是到 .com / .net / .org 就结束了
URL就是除了域名 还有后面的的/asl;dajs;dlfsdf.html 一长串资源路径

异步和同步：

举个例子：普通B/S模式（同步）AJAX技术（异步）

同步：提交请求->等待服务器处理->处理完毕返回 这个期间客户端浏览器不能干任何事

异步: 请求通过事件触发->服务器处理（这是浏览器仍然可以作其他事情）->处理完毕

同步就是你叫我去吃饭，我听到了就和你去吃饭；如果没有听到，你就不停的叫，直到我告诉你听到了，才一起去吃饭。

异步就是你叫我，然后自己去吃饭，我得到消息后可能立即走，也可能等到下班才去吃饭。

AJAX技术实现了异步，但不等于异步。

delete操作符

用于删除对象的某个属性。

返回值：对于所有情况都是true，除非属性是一个自己的不可配置属性，在这种情况下，非严格模式返回 false。

  • 如果你删除的属性在对象上不存在，那么delete将不会起作用，但仍会返回true
  • 如果 delete 操作符删除成功，则被删除的属性将从所属的对象上彻底消失。然后，如果该对象的原型链上有一个同名属性，则该对象会从原型链上继承该同名属性。（也就是说delete操作只会在自身的属性上起作用）
  • 当一个属性被设置为不可设置，delete操作将不会有任何效果，并且会返回false。如Object.defineProperty(obj, prop, descriptor)
  • 当你删除一个数组元素时，数组的 length 属性并不会变小。即便你删除了数组的最后一个元素也是如此。

顺序查找

  • 不需要有序

　　从数组的一端开始，顺序逐一扫描，依次与给定值k相比较，若相等则表示查找成功；若扫描结束仍没有找到等于k的元素，表示查找失败。

二分查找

  • 前提：有序数组

折半查找。
二分查找的基本思想是将n个元素分成大致相等的两部分，取a[n/2]与x做比较，如果x=a[n/2],则找到x,算法中止；如果xa[n/2],则只要在数组a的右半部搜索x.

可以用循环或递归

插值查找

  • 有序数组
  • 二分法的优化

改变了二分法的分法。

根据要查找的值在整个有序数组中所处的位置，让mid值的变化更靠近关键字key，这样也就间接地减少了比较次数。
mid=low+(key-a[low])/(a[high]-a[low])*(high-low)

Queue(队列)：

就像作家会把书分章分段落，程序员叶需要把代码分成模块，每个模块可以专注于一个功能。分割后，便于你增加减少或修改功能，同时又不会影响整个系统

模块化的优点?

可维护，避免命名空间被污染，可复用。

模块化的几种方法：
1.匿名闭包函数（立即执行函数）
2.全局引入
3.对象接口
4.……

webpack.config.js 各项意义及用法

入口(entry)

入口指示 webpack 应该使用哪个模块，来作为构建其内部依赖图的开始。进入入口起点后，webpack 会找出有哪些模块和库是入口起点（直接和间接）依赖的

出口(output)

output 属性告诉 webpack 在哪里输出它所创建的 bundles，以及如何命名这些文件。你可以通过在配置中指定一个 output 字段，来配置这些处理过程：

loader

loader让 webpack 能够去处理那些非 JavaScript 文件（webpack 自身只理解 JavaScript）。
loader 可以将所有类型的文件转换为 webpack 能够处理的有效模块，然后你就可以利用 webpack 的打包能力，对它们进行处理。

1.test属性：用于标识出应该被对应的 loader 进行转换的某个或某些文件
2.use属性：表示进行转换时，应该使用哪个 loader。

插件(plugins)

插件则可以用于执行范围更广的任务，从打包优化和压缩，到重新定义环境中的变量。

使用一个插件需要 require() ，然后把它添加到 plugins 数组中。
多数插件可以通过选项(option)自定义。

页面内边距

*{ padding: 0; margin: 0; }

要用css清除浏览器对页面的默认外边距、内边距。
此处*指网页里的所有元素。

更改默认的css盒子模型

box-sizing: border-box;

对div设置padding时会把div的宽高撑大。设置css样式box-sizing: border-box，元素的内边距和边框不再会增加它的宽度。

解释：

box-sizing 属性可以被用来调整这些表现:

  • content-box 是默认值。如果你设置一个元素的宽为100px，那么这个元素的内容区会有100px宽，并且任何边框和内边距的宽度都会被增加到最后绘制出来的元素宽度中。
  • border-box 告诉浏览器去理解你设置的边框和内边距的值是包含在width内的。也就是说，如果你将一个元素的width设为100px,那么这100px会包含其它的border和padding，内容区的实际宽度会是width减去border + padding的计算值。大多数情况下这使得我们更容易的去设定一个元素的宽高。

我觉得将所有的元素的box-sizing都设为border-box 能更好的区别margin和padding属性。

并排同行显示

display:inline-block

inline-block可以简单理解成inline+block，既可以设置width和height（block），又可以行内排列（inline）

用它来代替float 更加简单 也能实现相同的效果

元素垂直对齐方式

vertical-align: top/middle/bottom

即把元素的顶端与行中最高元素的顶端对齐/把元素放置在父元素的中部/把元素的顶端与行中最低的元素的顶端对齐。

图片位于颜色之上，若div宽长小于背景图，则最终看到的是背景图，若超出，则在背景图没覆盖到的地方显示背景颜色

div与span的区别

div为块元素 而span是行内元素
在不对两者设置css宽高时，div占用的位置是一行，而span占用的是文字内容的宽度。

虽然没有查到为什么，但在对两者设置左边框时，发现div的边框高度会比span的边框高度要长，而span的边框高度和文字高度一样。
