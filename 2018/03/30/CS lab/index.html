在学习数据结构的时候的一些零碎的笔记

ADT(abstract data type):抽象数据类型

数据类型的定义和它的实现是分开的。（例子：开车。表面的操作与内在的执行是分开的，同样，软件表面的操作暴露，而内在代码没有 ）

数据结构是什么？
数据结构（data structure）是计算机中存储、组织数据的方式。

Queue(队列)：

FIFO(first in first out) 队列只允许在后端（称为rear）进行插入操作，在前端（称为front）进行删除操作。
队列的操作方式和堆栈类似，唯一的区别在于队列只允许新数据在后端进行添加。
enqueue:插入
dequeue:删除

##Stack（栈）：
LIFO(last in first out) 堆箱子一样，最后堆的箱子在最上面 （push/pop/top(peek)/isEmpty/size/）只能允许在链接串列或阵列的一端（称为堆叠顶端指标，英语：top）进行加入数据（英语：push）和输出数据（英语：pop）的运算。（例子：浏览网站时后退）top()是将指针置于堆栈顶部 注意堆栈满了（full）和空（empty）的情况

ProAbs C++笔记：

枚举类型
好处：

  • C ++编译器对枚举类型进行了一些基本的检查。例如，如果没有类型转换，则不能将整数值分配给枚举变量，这有助于引起注意可能的错误，例如为枚举分配值超出范围
  • 使用枚举类型的￼Page Mode 程序更容易调试。编译器使枚举常量的名称可供调试器使用。

标量类型（scalar type）：枚举类型，字符和各种整数表示形成一个更普通的类型类，称为标量类型。当在C ++表达式中使用标量类型的值时，编译器会自动将其转换为用于表示该值的整数。这个规则的效果是，你可以对任何标量类型的值执行与整数的操作相同的操作。

空指针:在很多指针应用程序中，能够在指针变量中存储一个特殊值，这个值是很有用的，这个特殊值表示变量事实上并不指向任何有效数据，至少目前是这样。这样一个指针变量被称为空指针，它的值被赋值为地址0.为此可以使用称为NULL的特殊常量。

这个指针没有指向一块有意义的内存

record:即结构体

动态分配(Dynamic allocation):(静态分配、自动分配、动态分配)程序在运行期中分配使用内存。
堆(heap): 用new操作符来从堆里分配内存、
如int *ip = new int 会返回已保留一个整数的堆中的内存地址。
要在用完后释放内存，防止内存不足。用delete来进行释放
如delete ptr、delete[] arr

动态数组:动态数组的大小不确定，可根据数据多少调整

头文件：#include语句有两种方式包含头文件，分别是使用双引号” “与左右尖括号< >。其区别是（对于不是使用完全文件路径名的）头文件的搜索顺序不同：

使用双引号” “的头文件的搜索顺序：

  • 包含该#include语句的源文件所在目录；
  • 包含该#include语句的已经打开的头文件的逆序（因为头文件可以#include另一个头文件构成一个序列）；
  • 编译选项-I所指定的目录；
  • 环境变量INCLUDE所定义的目录。

使用左右尖括号< >的头文件的搜索顺序：

  • 编译选项-I所指定的目录
  • 环境变量INCLUDE所定义的目录
