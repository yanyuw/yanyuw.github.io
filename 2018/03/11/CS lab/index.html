3.5

ADT:数据类型的定义和它的实现是分开的。（例子：开车。表面的操作与内在的执行是分开的，同样，软件表面的操作暴露，而内在代码没有 ）
Queue(队列)：FIFO(first in first out) 队列只允许在后端（称为rear）进行插入操作，在前端（称为front）进行删除操作。
队列的操作方式和堆栈类似，唯一的区别在于队列只允许新数据在后端进行添加。
enqueue:插入
dequeue:删除
Stack（栈）：LIFO(last in first out) 堆箱子一样，最后堆的箱子在最上面 （push/pop/top(peek)/isEmpty/size/）只能允许在链接串列或阵列的一端（称为堆叠顶端指标，英语：top）进行加入数据（英语：push）和输出数据（英语：pop）的运算。（例子：浏览网站时后退）top()是将指针置于堆栈顶部 注意堆栈满了（full）和空（empty）的情况

3.8

数据结构是什么？
数据结构（data structure）是计算机中存储、组织数据的方式。

ProAbs C++：

枚举类型
好处：

  • C ++编译器对枚举类型进行了一些基本的检查。例如，如果没有类型转换，则不能将整数值分配给枚举变量，这有助于引起注意可能的错误，例如为枚举分配值超出范围
  • 使用枚举类型的程序更容易调试。编译器使枚举常量的名称可供调试器使用。

标量类型（scalar type）：枚举类型，字符和各种整数表示形成一个更普通的类型类，称为标量类型。当在C ++表达式中使用标量类型的值时，编译器会自动将其转换为用于表示该值的整数。这个规则的效果是，你可以对任何标量类型的值执行与整数的操作相同的操作。
